## Contributing

Thank you for your interest in contributing to ChatGuard! This guide will help you get started.

### Getting Started

1. **Fork** this repository on GitHub.
2. Create a new **branch** with a descriptive name.
3. Follow the project’s **coding style and conventions**.
4. **Test** your changes before submitting.
5. Use **[Conventional Commits](https://www.conventionalcommits.org/)** for your commit messages.
6. Open a **Pull Request** with a clear description.


## Help Publish on Google Play

ChatGuard is currently distributed only through GitHub Releases. Publishing it on **Google Play** is an important next step because:

* It improves **Android security and trust validation** for the app.
* It enables **automatic updates** and better discoverability through search.
* It increases **user confidence and adoption** across the community.

We are looking for help with the **Google Play publishing process**, including experience with the Play Console, release workflow, and handling submission steps. If you can contribute, please get in touch via this **[GitHub repository](https://github.com/amirroid/chatguard)**.

### Reporting Issues

* Use [GitHub Issues](https://github.com/amirroid/chatguard/issues) to report bugs or suggest features.
* Include your device model, Android version, and steps to reproduce when reporting a bug.

### Community

* When in doubt, open an issue to discuss your idea before investing time in a PR.
